What is OpS Crypt LTC?

OpS Crypt LTC is a non-custodial Litecoin wallet project: software that lets a person create real Litecoin keys, receive LTC, send LTC, and back up the wallet — without giving private keys to a bank or exchange.

1. Purpose of this project

The goal is a simple, focused product: Litecoin only (LTC mainnet). It is designed to feel like a lightweight crypto wallet for send and receive, with keys generated and stored under the user’s control.

The project includes the wallet source code, Android packaging support (APK), and this explanatory website in English and Portuguese.

2. What it does (explained simply)

Create a real wallet

When you create a wallet, the app generates a standard BIP39 seed phrase (usually 12 English words). From that seed it derives Litecoin addresses and private keys using industry paths (BIP84 SegWit and BIP44 Legacy).

Receive Litecoin

You show an address (or QR code). Someone sends LTC to that address on the Litecoin network. The app reads public blockchain APIs to show balance and history.

Send Litecoin

You enter a destination address and amount. The transaction is signed on the device with your private key, then broadcast through public Litecoin APIs. Servers never receive your seed or private keys for signing.

Backup and restore

You can export an encrypted backup file (.opsjson) protected by a PIN, import a seed phrase, or import a WIF private key. You can also restore from the encrypted backup upload.

3. Security model (plain language)

  • Non-custodial: you hold the keys; the project author does not hold your coins.
  • PIN encryption: the seed/keys on the device are stored encrypted (AES-256-GCM with PBKDF2).
  • Local signing: private keys are used only on the device to sign transactions.
  • Public APIs: only public addresses and already-signed transactions are sent to network APIs.
  • Your responsibility: if you lose the seed and backup, nobody can recover the funds — not even the author.
Never type your seed phrase on unknown websites. Never share your PIN, seed, or WIF. Always verify the address before sending real value. Start with small amounts.

4. Compatibility with other wallets

The seed uses the common BIP39 standard. That means you can often import the same phrase into other wallets that support Litecoin.

WalletWorks with OpS seed/LTC?Explanation
Trust Wallet Yes Import the 12 words, enable Litecoin. Legacy path often matches.
Exodus and similar multi-coin wallets Usually yes Import recovery phrase if Litecoin is supported.
Electrum-LTC Via WIF Export WIF from OpS Crypt and import there.
MetaMask No MetaMask is for Ethereum/EVM chains only. It does not support Litecoin.

O que é o OpS Crypt LTC?

OpS Crypt LTC é um projeto de carteira Litecoin não-custodial: um software que permite criar chaves reais de Litecoin, receber LTC, enviar LTC e fazer backup — sem entregar a chave privada a um banco ou corretora.

1. Objetivo do projeto

O foco é um produto simples: somente Litecoin (rede mainnet). A ideia é uma carteira leve para enviar e receber, com chaves geradas e guardadas sob controle do usuário.

O repositório inclui o código da carteira, suporte a empacotamento Android (APK) e este site explicativo em inglês e português.

2. O que ele faz (explicado de forma simples)

Criar uma carteira real

Ao criar a carteira, o app gera uma frase-semente BIP39 (em geral 12 palavras em inglês). A partir dela deriva endereços e chaves Litecoin com caminhos padrão (BIP84 SegWit e BIP44 Legacy).

Receber Litecoin

Você mostra um endereço (ou QR). Alguém envia LTC para esse endereço na rede Litecoin. O app consulta APIs públicas da blockchain para mostrar saldo e histórico.

Enviar Litecoin

Você informa o endereço de destino e o valor. A transação é assinada no aparelho com a chave privada e depois transmitida por APIs públicas. Os servidores não recebem a seed nem a chave privada para assinar.

Backup e restauração

É possível exportar um arquivo de backup criptografado (.opsjson) com PIN, importar frase-semente, importar WIF, ou restaurar pelo upload do backup.

3. Modelo de segurança (linguagem clara)

  • Não-custodial: você detém as chaves; o autor do projeto não guarda suas moedas.
  • Criptografia com PIN: seed/chaves no aparelho ficam cifradas (AES-256-GCM + PBKDF2).
  • Assinatura local: a chave privada só é usada no dispositivo.
  • APIs públicas: só endereço público e transação já assinada vão para a rede.
  • Sua responsabilidade: se perder a seed e o backup, ninguém recupera os fundos — nem o autor.
Nunca digite a seed em sites desconhecidos. Não compartilhe PIN, seed ou WIF. Confira sempre o endereço antes de enviar valor real. Comece com valores pequenos.

4. Compatibilidade com outras carteiras

A seed usa o padrão comum BIP39. Por isso, em muitos casos, a mesma frase pode ser importada em outras carteiras que suportam Litecoin.

Electrum-LTC Via WIF Exporte o WIF no OpS Crypt e importe lá.
